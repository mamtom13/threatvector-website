PHARMING & DNS HIJACKING

SEVERITY: HIGH
What is Pharming?

Pharming is the manipulation of DNS resolution to redirect users from legitimate websites to fraudulent ones without their knowledge. Unlike phishing, pharming does not require user interaction with a malicious link — it poisons the DNS layer itself.

How it works

Attackers compromise DNS resolvers, routers, or ISP infrastructure to redirect queries for legitimate domains to attacker-controlled servers. Victims type the correct URL but are silently redirected. Pharming is particularly dangerous because the URL bar appears correct.

Key Indicators

▸ DNS responses returning unexpected IP addresses for known-good domains
▸ Router DNS settings changed to attacker-controlled resolvers
▸ ISP-level DNS responses inconsistent with authoritative nameservers
▸ SSL certificate mismatch warnings for previously trusted sites
▸ Multiple users at the same network location experiencing identical redirects
▸ BGP route hijacking causing traffic interception at network level

How to Report

Document the affected domain, expected IP vs. observed IP, DNS resolver returning incorrect results, network location (ISP, ASN), and timestamps. Packet captures showing DNS response manipulation are essential.
