DGA & FAST-FLUX DOMAINS

SEVERITY: CRITICAL
What are DGA & Fast-Flux?

Domain Generation Algorithms (DGA) are algorithms embedded in malware that automatically generate large numbers of domain names as potential C2 rendezvous points. Fast-flux DNS rapidly rotates the IP addresses associated with a domain to make takedown extremely difficult.

How it works

DGA malware generates hundreds of domain names daily using a seed value (often the current date). Only the attacker knows which domains are registered — the malware tries each until one resolves. Fast-flux networks use compromised hosts as proxies, changing DNS records every 60–300 seconds.

Key Indicators

▸ Domains with pseudo-random, algorithmically-generated names
▸ High volume of NXDOMAIN responses followed by a single successful resolution
▸ DNS TTL values of 60 seconds or less (fast-flux indicator)
▸ Single domain resolving to 10+ different IPs within minutes
▸ IP addresses belonging to residential ISPs (compromised proxy hosts)
▸ Domain registration dates matching malware sample compilation dates

How to Report

Provide the identified domain names, suspected DGA algorithm or family, malware samples generating the domains, passive DNS records showing IP rotation, and any predicted future domains if the algorithm has been reverse-engineered.
